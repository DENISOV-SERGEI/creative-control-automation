# Отчёт по проекту автоматизации «Контроль сроков годности рекламных креативов»

## 1. Цель автоматизации
Исключить показ просроченных баннеров на маркетплейсах и лендингах, сократить ручной труд маркетолога по мониторингу сроков акций.

## 2. Результат внедрения
- Просроченные креативы автоматически деактивируются (статус Active = FALSE).
- Маркетолог получает уведомления в Telegram:
  - за 3 дня до окончания (льготный период);
  - в момент отключения.
- Ведётся полный лог аудита (таблица Audit Log).
- При сбоях администратор получает оповещение, ошибка записывается в отдельный лист.

## 3. Архитектура решения
- **Платформа**: n8n (self-hosted).
- **Интеграции**: Google Sheets (чтение/обновление/добавление строк), Telegram Bot API.
- **Триггер**: Schedule Trigger (ежедневно в 11:00 МСК).
- **Логика**:
  - Параллельная проверка двух условий:
    - `End_date < сегодня` → отключение;
    - `0 < (End_date - сегодня) <= 3 дня` → предупреждение.
  - Обработчик ошибок: отдельный workflow с Error Trigger.
- **Обработка дат**: библиотека Luxon (n8n) + кастомные выражения JavaScript.

## 4. Использованные сервисы и их роль
| Сервис | Роль |
|--------|------|
| Google Sheets (creatives_monitor) | Хранение креативов, их статусов, дат окончания, логов |
| Google Sheets (Audit Log) | Хранение истории всех событий (deactivated / warning) |
| Google Sheets (Ошибки) | Логирование сбоев автоматизации |
| Telegram Bot | Уведомления маркетологу и администратору |
| n8n | Оркестрация workflow, выполнение условий, преобразование данных |

## 5. Ключевые технические особенности
- Использование `DateTime.fromFormat` для парсинга дат в формате `ДД.ММ.ГГГГ`.
- Сравнение дат с точностью до начала дня (`startOf('day')`).
- Параллельное ветвление данных через два узла IF, получающих один массив строк.
- Автоматическое обновление колонок `Active` и `log` при отключении.
- Динамическое форматирование сообщений Telegram (HTML).
- Обработка ошибок с перехватом в отдельном workflow, отправка детализированного уведомления.

## 6. Результаты тестирования
| Сценарий | Результат |
|----------|-----------|
| Креатив с датой вчера, Active=TRUE | Active=FALSE, log заполнен, Telegram отправлен, запись в Audit Log (deactivated) |
| Креатив с датой через 1 день, Active=TRUE | Telegram-предупреждение, запись в Audit Log (warning), таблица не изменена |
| Креатив с датой через 4 дня, Active=TRUE | Нет действий |
| Креатив с Active=FALSE (любая дата) | Нет действий |
| Ошибка доступа к Google Sheets | Сработал Error Handler: Telegram админу, запись в таблицу «Ошибки» |

## 7. Метрики эффективности
| Показатель | До автоматизации | После автоматизации |
|------------|------------------|----------------------|
| Показы просроченных акций | периодически | 0 |
| Ручная работа маркетолога (в день) | 30–60 мин | 0 мин |
| Время реакции на истечение срока | от часов до дней | мгновенно (в 11:00) |
| Наличие истории отключений | отсутствует | полный аудит |
| Оповещение о сбоях | нет | Telegram + лог ошибок |
